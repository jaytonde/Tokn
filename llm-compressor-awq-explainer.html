LLM Compressor: AWQ Quantization Code Walkthrough

Top-down map of how vllm-project/llm-compressor quantizes a model, where AWQ finds activation-weighted scales, and where the actual qparams / quantized behavior are implemented.

One-Line Mental Model

AWQModifier Finds per-channel smoothing scales from calibration activations and temporary quantized-weight error.
QuantizationModifier Attaches quantization schemes, observers, qparams, and frozen quantization status to target modules.
compressed_tensors Provides the low-level quantization lifecycle, fake/forward quantization, and compressed checkpoint packing.
AWQ in this repo is a transform, not the final packer. It rescales model weights to protect salient activation channels, then the normal quantization path computes qparams and saves compressed tensors.

User-Level Flow

recipe = [
    AWQModifier(duo_scaling="both"),
    QuantizationModifier(
        ignore=["lm_head"],
        scheme="W4A16_ASYM",
        targets=["Linear"],
    ),
]

oneshot(model=model, dataset=ds, recipe=recipe, ...)
model.save_pretrained(SAVE_DIR, save_compressed=True)

AWQ Scale Search

Calibration activations
X
+
Layer weights
W
->
Choose scale
s
Loss(s) = || Q(W * s) * (X / s) - W * X ||

The repo implements this directly in AWQModifier._compute_best_scale. It builds x_mean from cached absolute activations. With duo scaling enabled, it also computes w_mean from normalized weight magnitudes.

PartImplementationMeaning
Activation salience _setup_activation_cache_hooks Forward hook collects abs(input), flattens tokens, then stores per-channel sum and count.
Weight contribution _compute_layer_means Normalizes weight chunks according to tensor/channel/group/block quantization strategy, then averages per input channel.
Candidate scale scales = x_mean.pow(ratio) or x_mean.pow(ratio) / w_mean.pow(1-ratio) Grid search tests ratios across n_grid. duo_scaling="both" tries activation-only and activation+weight versions.
Error test forward_quantize(..., "weight", ...) then _compute_loss Temporarily quantizes scaled weights, re-runs the parent module, and picks the scale with lowest output MSE.
Apply result _apply_smoothing Multiplies downstream/balance layer weights by s and divides the smooth layer weight/bias by s.

Where The Actual Quantization Happens

The qparams are not computed in AWQ. They are computed through observers in src/llmcompressor/modifiers/quantization/calibration.py.

apply_quantization_config
Imported from compressed_tensors and called by QuantizationMixin.initialize_quantization.

Attaches weight/input/output observers to quantized modules.
stats
observe()
Feeds module weights or activations into observers.

update_qparams()
Calls observer.get_qparams() and stores scale / zero point on the module.
runtime/save
forward_quantize / compress_model
Low-level quantized math and packing come from compressed_tensors.

Bottom Line

If you are looking for "AWQ finding salient weights", read AWQModifier._setup_activation_cache_hooks, _compute_layer_means, and _compute_best_scale. If you are looking for "actual quantization", read QuantizationModifier.on_sequential_epoch_end and calibration.update_qparams, then follow the imported compressed_tensors functions.
